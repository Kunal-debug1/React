What is React?

React is a JavaScript library for building user interfaces. Created by Meta, it lets you build complex UIs from small, isolated pieces called components. React takes care of efficiently updating and rendering the right components when your data changes.

🏪Real World Example
Think of React like a restaurant menu system

Each dish is a component — it has its own look and ingredients (props). The kitchen's order list is state — it updates as orders come in. When a new order arrives, only the affected display updates — not the entire restaurant. That's React's efficient rendering.

Why React?

♻️
Reusable Components
Build once, use everywhere. A Button component you write once works across your entire app.
⚡
Fast Updates
React's Virtual DOM updates only what changed, not the whole page. Blazing fast.
🌊
Declarative
You describe WHAT the UI should look like, React figures out HOW to make it happen.
🌍
Huge Ecosystem
Next.js, React Native, thousands of libraries — React skills transfer everywhere.

Home/Components & JSX
🧩 Core Concepts

Components & JSX

Components are the building blocks of React. JSX is the syntax that makes writing them feel natural.

Your First Component

A React component is just a JavaScript function that returns JSX (which looks like HTML). Component names must start with a capital letter.

Button.jsx
// A simple React component
function Button() {
  return (
    <button className="btn">
      Click me!
    </button>
  );
}

export default Button;
🏪Real World Example
A product card on Amazon

Every product you see on Amazon is the same <ProductCard /> component, just rendered with different data. One component, hundreds of products. That's the power of reusability.

Nesting Components

Components can include other components. This is how you build complex UIs from simple pieces.

App.jsx
import Navbar from './Navbar';
import ProductCard from './ProductCard';

function App() {
  return (
    <div>
      <Navbar />           {/* Our custom component */}
      <main>
        <ProductCard />    {/* Reusable, nested */}
        <ProductCard />
      </main>
    </div>
  );
}

export default App;

JSX Rules

📌
Remember These Rules

1. Return a single root element — wrap multiple elements in <div> or <> (Fragment).
2. Close all tags — <br /> not <br>.
3. Use className instead of class.
4. Use htmlFor instead of for on labels.
5. JavaScript expressions go inside { } curly braces.

JSX Example
function Profile() {
  const name = 'Rahul';
  const age = 25;

  return (
    <>                          {/* Fragment — no extra div */}
      <h1>Hello, {name}!</h1>   {/* { } embeds JS */}
      <p>Age: {age}</p>
      <p>Next year: {age + 1}</p> {/* Expressions work too */}
      <img className="avatar" alt="Profile" />
    </>
  );
}

Styles in JSX

There are two main ways to style components:

Styling.jsx
// 1. Using className (external CSS)
<button className="btn-primary">Click</button>

// 2. Inline styles (double curly braces)
<button style={{ backgroundColor: 'blue', color: 'white' }}>
  Click
</button>
// Note: camelCase for CSS properties (backgroundColor, not background-color)
Home/Props
🧩 Core Concepts

Props

Props (short for "properties") are how you pass data from a parent component to a child. They make components dynamic and reusable.

Passing Props

Props are passed like HTML attributes. In the child component, you receive them as a function parameter called props (or you can destructure them directly).

App.jsx
// PARENT — passes data down
function App() {
  return (
    <div>
      <ProductCard
        name="iPhone 15"
        price={79999}
        inStock={true}
      />
    </div>
  );
}

// CHILD — receives data (destructured)
function ProductCard({ name, price, inStock }) {
  return (
    <div className="card">
      <h2>{name}</h2>
      <p>₹{price}</p>
      <p>{inStock ? 'In Stock ✅' : 'Out of Stock ❌'}</p>
    </div>
  );
}
📱Real World Example
Flipkart Product Listings

Every product card on Flipkart is the same component. The name, image, price, rating, and discount are all props passed from an array of product data. One component template, thousands of unique products rendered.

Default Props

Button.jsx
function Button({ label = 'Click Me', color = 'blue', onClick }) {
  return (
    <button
      onClick={onClick}
      style={{ background: color }}
    >
      {label}
    </button>
  );
}

// Usage — label defaults to 'Click Me' if not passed
<Button />
<Button label="Buy Now" color="green" />

The children Prop

The special children prop lets you pass JSX content between a component's opening and closing tags — like a slot in a template.

Card.jsx
function Card({ title, children }) {
  return (
    <div className="card">
      <h3>{title}</h3>
      <div className="card-body">
        {children}  {/* Renders whatever is inside <Card>...</Card> */}
      </div>
    </div>
  );
}

// Usage:
<Card title="Order Summary">
  <p>2 items in your cart</p>
  <button>Checkout</button>
</Card>
📌
Props are Read-Only!

A component must never modify its own props. Props flow one way — down from parent to child. If you need to change something, that's what state is for.

Home/State
🧠 Core Concepts

State & useState

State is a component's memory. When state changes, React re-renders the component to reflect the new data.

What is State?

Unlike props, state is private to the component and can change over time. Think of it as variables that React watches — when they change, the UI updates automatically.

🛒Real World Example
Shopping Cart Count

The cart icon showing "3 items" uses state. When you add a product, the cart count state changes, and React automatically updates the badge number on screen. No manual DOM manipulation needed.

useState Hook

Counter.jsx
import { useState } from 'react';

function Counter() {
  // [currentValue, setterFunction] = useState(initialValue)
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>+</button>
      <button onClick={() => setCount(count - 1)}>-</button>
      <button onClick={() => setCount(0)}>Reset</button>
    </div>
  );
}

State with Objects

UserProfile.jsx
function UserProfile() {
  const [user, setUser] = useState({
    name: 'Priya',
    age: 24,
    city: 'Mumbai'
  });

  function updateCity() {
    // ALWAYS spread the old state — never mutate directly!
    setUser({ ...user, city: 'Delhi' });
  }

  return (
    <div>
      <p>{user.name} lives in {user.city}</p>
      <button onClick={updateCity}>Move to Delhi</button>
    </div>
  );
}
🚫
Never Mutate State Directly

user.city = 'Delhi' ❌ — This doesn't trigger a re-render!
setUser({ ...user, city: 'Delhi' }) ✅ — This creates new state and triggers re-render.

Lifting State Up

When two components need to share the same state, move (lift) it to their closest common parent, then pass it down via props.

App.jsx
function App() {
  // State lives in parent — shared by both children
  const [cartCount, setCartCount] = useState(0);

  return (
    <div>
      <Navbar cartCount={cartCount} />
      <ProductList
        onAddToCart={() => setCartCount(cartCount + 1)}
      />
    </div>
  );
}
Home/Events
🎯 Core Concepts

Events & Interactivity

React handles events declaratively. You attach handlers directly to JSX elements — no addEventListener needed.

Event Handlers

Events.jsx
function LoginForm() {
  function handleClick() {
    alert('Button clicked!');
  }

  function handleMouseOver(e) {
    console.log('Hovered!', e.target);  // e = event object
  }

  return (
    <button
      onClick={handleClick}          {/* pass ref, not call */}
      onMouseOver={handleMouseOver}
    >
      Login
    </button>
  );
}
⚠️
Pass the Function, Don't Call It

onClick={handleClick} ✅ — passes the function reference
onClick={handleClick()} ❌ — calls it immediately on render!

Handling Input

Search.jsx
import { useState } from 'react';

function SearchBar() {
  const [query, setQuery] = useState('');

  return (
    <div>
      <input
        type="text"
        value={query}
        placeholder="Search products..."
        onChange={(e) => setQuery(e.target.value)}
      />
      <p>You typed: {query}</p>
    </div>
  );
}
🔍Real World Example
Google Search Autocomplete

Every keystroke fires an onChange event. React updates the query state, which triggers a re-render showing live suggestions. State + Events working together.

Common Events

onClick
Button clicks, div clicks, any click interaction
onChange
Input value changes, checkbox toggles, select changes
onSubmit
Form submission — always use e.preventDefault()
onKeyDown
Keyboard key press (check e.key === 'Enter')
onFocus / onBlur
Input gains / loses focus
onMouseEnter
Mouse enters element — for tooltips, hover effects
Home/Conditional Rendering
🧩 Core Concepts

Conditional Rendering

Show different UI based on conditions — using regular JavaScript inside JSX.

Three Ways to Conditionally Render

Conditional.jsx
function Dashboard({ isLoggedIn, isAdmin, notifications }) {
  // METHOD 1: if/else — best for complex conditions
  if (!isLoggedIn) {
    return <LoginPage />;
  }

  return (
    <div>
      {/* METHOD 2: Ternary — inline, two branches */}
      {isAdmin
        ? <AdminPanel />
        : <UserPanel />
      }

      {/* METHOD 3: && — show or show nothing */}
      {notifications.length > 0 && (
        <NotificationBadge count={notifications.length} />
      )}
    </div>
  );
}
🔐Real World Example
Netflix Login/Home Screen

When you open Netflix, it checks isLoggedIn. If false → show <LoginScreen />. If true → show <HomePage />. Same app, different UI based on state. That's conditional rendering.

⚠️
&& Pitfall with Numbers

{count && <Badge />} will render 0 when count is 0 (not nothing)!
Fix: {count > 0 && <Badge />} or {Boolean(count) && <Badge />}

Home/Lists & Keys
📋 Core Concepts

Lists & Keys

Render arrays of data as lists of components using .map() — the most used pattern in React.

Rendering a List

ProductList.jsx
const products = [
  { id: 1, name: 'iPhone 15', price: 79999 },
  { id: 2, name: 'Samsung S24', price: 74999 },
  { id: 3, name: 'OnePlus 12', price: 64999 },
];

function ProductList() {
  return (
    <ul>
      {products.map(product => (
        <li key={product.id}>  {/* key is required! */}
          {product.name} — ₹{product.price}
        </li>
      ))}
    </ul>
  );
}

Why Keys Matter

Keys help React identify which items changed, were added, or removed. React uses keys to match old and new list items efficiently.

🚫
Don't Use Array Index as Key

key={index} causes bugs when items are reordered or deleted. Always use a unique, stable ID from your data — like a database ID or UUID.

Filtering Lists

FilteredList.jsx
function ProductList({ searchQuery }) {
  // Filter first, then map — chain them!
  const filtered = products
    .filter(p => p.name.toLowerCase().includes(searchQuery.toLowerCase()))
    .map(p => <ProductCard key={p.id} product={p} />);

  return (
    <div>
      {filtered.length > 0
        ? filtered
        : <p>No products found.</p>}
    </div>
  );
}
Home/useEffect
⚡ Hooks

useEffect

useEffect lets you synchronize a component with an external system — fetching data, setting up subscriptions, timers, and DOM manipulation.

The Dependency Array

useEffect patterns
import { useEffect, useState } from 'react';

// 1. Runs after EVERY render
useEffect(() => { console.log('rendered'); });

// 2. Runs ONCE on mount (empty array)
useEffect(() => { fetchUserData(); }, []);

// 3. Runs when userId changes
useEffect(() => { fetchUserData(userId); }, [userId]);

// 4. With cleanup (runs before unmount)
useEffect(() => {
  const timer = setInterval(() => setTime(t => t + 1), 1000);
  return () => clearInterval(timer); // ← cleanup function
}, []);

Fetching Data

UserProfile.jsx
function UserProfile({ userId }) {
  const [user, setUser] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    setLoading(true);
    fetch(`https://api.example.com/users/${userId}`)
      .then(res => res.json())
      .then(data => { setUser(data); setLoading(false); })
      .catch(err => { setError(err.message); setLoading(false); });
  }, [userId]); // Re-fetch when userId changes

  if (loading) return <p>Loading...</p>;
  if (error) return <p>Error: {error}</p>;

  return <h1>Hello, {user.name}!</h1>;
}
📰Real World Example
News Feed Loading

When you open Twitter/X, a useEffect fires on mount, calls the API, and populates your feed. When you scroll to a new page, the [page] dependency triggers another fetch. Loading spinners are set via loading state.

📌
The Cleanup Function

Always clean up effects that create subscriptions, timers, or event listeners. Return a cleanup function from useEffect — React calls it when the component unmounts or before re-running the effect.

Home/useRef & useCallback
🪝 Hooks

useRef & useCallback

useRef gives you a mutable value that persists across renders without triggering them. useCallback memoizes functions so they don't get recreated every render.

useRef — Two Use Cases

UseRef.jsx
import { useRef, useState, useEffect } from 'react';

function VideoPlayer() {
  // USE CASE 1: Accessing DOM elements directly
  const videoRef = useRef(null);

  function handlePlay() {
    videoRef.current.play(); // Direct DOM access!
  }

  // USE CASE 2: Store a value that doesn't trigger re-render
  const renderCount = useRef(0);
  renderCount.current += 1; // Changes without causing re-render

  return (
    <div>
      <video ref={videoRef} src="movie.mp4" />
      <button onClick={handlePlay}>Play</button>
    </div>
  );
}

useCallback — Memoize Functions

UseCallback.jsx
import { useCallback, useState } from 'react';

function ProductList() {
  const [cart, setCart] = useState([]);

  // Without useCallback: new function reference every render
  // With useCallback: same reference unless cart changes
  const handleAddToCart = useCallback((product) => {
    setCart(prev => [...prev, product]);
  }, []); // empty deps = never recreate

  return products.map(p => (
    <ProductCard
      key={p.id}
      product={p}
      onAdd={handleAddToCart} {/* stable ref = no child re-render */}
    />
  ));
}
💡
When to Use useCallback

Use it when passing callbacks to child components wrapped in React.memo, or as dependencies in useEffect. Don't overuse it — premature optimization adds complexity without always helping.

Home/useMemo & Performance
⚡ Performance

useMemo & Performance

useMemo caches the result of an expensive computation so it only recalculates when its dependencies change.

useMemo

ExpensiveList.jsx
import { useMemo, useState } from 'react';

function ProductCatalog({ products, searchQuery }) {
  // Without useMemo: filters 10,000 products every render
  // With useMemo: only recalculates when products or query changes
  const filteredProducts = useMemo(() => {
    console.log('Filtering products...'); // See how often this runs
    return products.filter(p =>
      p.name.toLowerCase().includes(searchQuery.toLowerCase())
    );
  }, [products, searchQuery]);

  return filteredProducts.map(p => <ProductCard key={p.id} product={p} />);
}

React.memo — Prevent Child Re-renders

ProductCard.jsx
import { memo } from 'react';

// Only re-renders if its props actually change
const ProductCard = memo(function ProductCard({ product, onAdd }) {
  console.log('ProductCard rendered');
  return (
    <div>
      <h3>{product.name}</h3>
      <button onClick={() => onAdd(product)}>Add to Cart</button>
    </div>
  );
});
⚠️
Don't Over-Optimize

Memoization has a cost (memory + comparison). Profile first with React DevTools. Only add useMemo/memo when you have a measured performance problem, not "just in case".

Home/Custom Hooks
🪝 Advanced

Custom Hooks

Custom hooks let you extract and reuse stateful logic across components — the React way of code sharing.

What is a Custom Hook?

A custom hook is just a function that starts with use and can call other hooks. Extract repeated logic once, use it everywhere.

🔁Real World Example
Fetch data in every page component

Without a custom hook, you'd copy-paste the same loading/error/data state pattern in every component that fetches data. A useFetch hook extracts this logic once — all your components stay clean.

useFetch Hook

hooks/useFetch.js
import { useState, useEffect } from 'react';

export function useFetch(url) {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    let cancelled = false;
    setLoading(true);

    fetch(url)
      .then(res => res.json())
      .then(json => { if (!cancelled) setData(json); })
      .catch(err => { if (!cancelled) setError(err.message); })
      .finally(() => { if (!cancelled) setLoading(false); });

    return () => { cancelled = true; }; // cleanup
  }, [url]);

  return { data, loading, error };
}

// Usage — so clean!
function UserPage({ userId }) {
  const { data: user, loading, error } = useFetch(
    `/api/users/${userId}`
  );

  if (loading) return <Spinner />;
  if (error) return <Error message={error} />;
  return <h1>Hello {user.name}</h1>;
}

useLocalStorage Hook

hooks/useLocalStorage.js
function useLocalStorage(key, initialValue) {
  const [value, setValue] = useState(() => {
    const stored = localStorage.getItem(key);
    return stored ? JSON.parse(stored) : initialValue;
  });

  function setStoredValue(newValue) {
    setValue(newValue);
    localStorage.setItem(key, JSON.stringify(newValue));
  }

  return [value, setStoredValue];
}

// Usage — persists across browser sessions!
const [theme, setTheme] = useLocalStorage('theme', 'light');
Home/Context API
🌐 Advanced

Context API

Context lets you pass data through the component tree without manually threading props through every level — solving "prop drilling".

The Problem: Prop Drilling

App (theme="dark") → Layout → Sidebar → Button (finally uses theme!)
↑ Layout and Sidebar don't use theme — they just pass it along. That's prop drilling. 😫

The Solution: Context

ThemeContext.jsx
import { createContext, useContext, useState } from 'react';

// STEP 1: Create the context
const ThemeContext = createContext('light');

// STEP 2: Create a Provider component
export function ThemeProvider({ children }) {
  const [theme, setTheme] = useState('light');
  const toggleTheme = () => setTheme(t => t === 'light' ? 'dark' : 'light');

  return (
    <ThemeContext.Provider value={{ theme, toggleTheme }}>
      {children}
    </ThemeContext.Provider>
  );
}

// STEP 3: Custom hook to consume context easily
export function useTheme() {
  return useContext(ThemeContext);
}

// USAGE: Wrap your app in the Provider
function App() {
  return (
    <ThemeProvider>
      <Layout />
    </ThemeProvider>
  );
}

// USAGE: Consume from ANYWHERE in the tree
function Button() {
  const { theme, toggleTheme } = useTheme(); // no prop needed!
  return <button onClick={toggleTheme}>Switch to {theme}</button>;
}
🌙Real World Example
Dark Mode Toggle (like Twitter/Notion)

The dark/light toggle sets a theme value in Context. Every component (Sidebar, Navbar, Cards, Buttons) reads from the same Context — all update simultaneously. Without Context, you'd need to pass theme as a prop through every single component.

⚠️
Context Re-renders All Consumers

Every component that calls useContext() re-renders when the Context value changes. For frequently changing values (like mouse position), split contexts or use a state manager like Zustand/Redux instead.

Home/useReducer
🌐 Advanced

useReducer

useReducer is an alternative to useState for complex state logic with multiple sub-values or when the next state depends on the previous one.

useState vs useReducer

SituationuseStateuseReducer
Simple toggle, counter✅ Use thisOverkill
Multiple related state fieldsGets messy✅ Use this
Next state depends on previousTricky✅ Use this
Complex actions (add/remove/update)Scattered✅ Use this
CartReducer.jsx
import { useReducer } from 'react';

// Reducer: pure function — takes state + action, returns new state
function cartReducer(state, action) {
  switch (action.type) {
    case 'ADD_ITEM':
      return { ...state, items: [...state.items, action.item], count: state.count + 1 };
    case 'REMOVE_ITEM':
      return { ...state, items: state.items.filter(i => i.id !== action.id), count: state.count - 1 };
    case 'CLEAR':
      return { items: [], count: 0 };
    default:
      return state;
  }
}

function Cart() {
  const [cart, dispatch] = useReducer(cartReducer, { items: [], count: 0 });

  return (
    <div>
      <p>{cart.count} items in cart</p>
      <button onClick={() => dispatch({ type: 'ADD_ITEM', item: { id: 1, name: 'Shoes' } })}>
        Add Shoes
      </button>
      <button onClick={() => dispatch({ type: 'CLEAR' })}>Clear Cart</button>
    </div>
  );
}
Home/Forms
📝 Advanced

Forms & Controlled Inputs

In React, form inputs are "controlled" — React state is the single source of truth for the input value.

Controlled Form

SignupForm.jsx
import { useState } from 'react';

function SignupForm() {
  const [form, setForm] = useState({ name: '', email: '', password: '' });
  const [errors, setErrors] = useState({});

  function handleChange(e) {
    const { name, value } = e.target;
    setForm(prev => ({ ...prev, [name]: value }));
  }

  function validate() {
    const errs = {};
    if (!form.email.includes('@')) errs.email = 'Invalid email';
    if (form.password.length < 8) errs.password = 'Min 8 characters';
    return errs;
  }

  function handleSubmit(e) {
    e.preventDefault(); // prevent page reload
    const errs = validate();
    if (Object.keys(errs).length > 0) { setErrors(errs); return; }
    console.log('Submitted:', form);
  }

  return (
    <form onSubmit={handleSubmit}>
      <input name="name" value={form.name} onChange={handleChange} placeholder="Name" />
      <input name="email" value={form.email} onChange={handleChange} placeholder="Email" />
      {errors.email && <span className="error">{errors.email}</span>}
      <button type="submit">Sign Up</button>
    </form>
  );
}
Home/Async & APIs
🌐 Advanced

Async, APIs & Loading States

Real apps fetch data. Learn the complete pattern for loading, error handling, and displaying API data in React.

The Complete Async Pattern

NewsPage.jsx
import { useState, useEffect } from 'react';

function NewsPage() {
  const [articles, setArticles] = useState([]);
  const [status, setStatus] = useState('idle'); // idle|loading|success|error
  const [error, setError] = useState(null);

  useEffect(() => {
    async function loadArticles() {
      setStatus('loading');
      try {
        const res = await fetch('https://newsapi.org/v2/top-headlines?country=in');
        if (!res.ok) throw new Error('Failed to fetch');
        const data = await res.json();
        setArticles(data.articles);
        setStatus('success');
      } catch (err) {
        setError(err.message);
        setStatus('error');
      }
    }
    loadArticles();
  }, []);

  if (status === 'loading') return <LoadingSpinner />;
  if (status === 'error') return <ErrorMessage message={error} />;

  return (
    <div>
      {articles.map(article => (
        <ArticleCard key={article.url} article={article} />
      ))}
    </div>
  );
}
💡
Use a String Status (not multiple booleans)

Instead of isLoading + isError + isSuccess, use a single status string: 'idle' | 'loading' | 'success' | 'error'. It's impossible to have contradictory states (like isLoading=true and isSuccess=true at once).

Hooks Cheatsheet

Every React hook, when to use it, and a one-liner example.

Built-in Hooks

useState
Store and update component state. const [val, setVal] = useState(initial)
useEffect
Run side effects after render. Fetch data, timers, subscriptions.
useContext
Read from a Context. const theme = useContext(ThemeContext)
useRef
DOM references + mutable values that don't cause re-renders.
useMemo
Cache expensive computed values. const val = useMemo(() => compute(), [deps])
useCallback
Cache function references. Use with React.memo to prevent child re-renders.
useReducer
Complex state with actions. Alternative to useState for state machines.
useId
Generate unique IDs for accessibility (htmlFor / id pairs).
useTransition
Mark state updates as non-urgent — keep UI responsive during heavy updates.
useDeferredValue
Defer re-rendering of a slow part of UI — show stale data while fresh loads.

  );
}

Golden Rules

1

Single Responsibility

Each component should do one thing well. If it's doing too much, split it into smaller components.

2

Lift State to Lowest Common Ancestor

Don't put state at the top of your tree "just in case". Place it as low as possible — only lift when siblings need to share it.

3

Don't Sync State — Derive It

If a value can be computed from existing state/props, don't store it in state. Just calculate it during render.

4

Use Keys Correctly

Stable, unique IDs from your data — never array indexes for dynamic lists.

5

Extract Logic Into Custom Hooks

If a component has more than 2-3 useEffects or complex state logic, extract it into a custom hook.
